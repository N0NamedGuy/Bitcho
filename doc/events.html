Events Overview

An event is triggered when a message is received from the IRC server, or it is explicitly triggered by, for instance, a call to the PuginBase.dispatch_event() method.
Each event has its type and parameters. For instance, the event join has as its parameters the user who joined the channel, and the joined channel.
Plugins, as said earlier, have their own ability to trigger events. All they need to do is call the dispatch_event method inherited from the PluginBase class.

Event Reference

numeric

event_num, nick, params, msg
Triggered whenever a numeric reply is received. If you do not understand this, it is better that you read the IRC RFC.
nick - The nick in the reply (depends on the numeric message)
params - The reply parameters
msg - The reply message

ping

msg
Triggered whenever a ping message is received. Most plugins wouldn'tcare to handle this event, but if you want to do so, have a go at it.
msg - The message to be sent on PONG command.

channel_msg

user, channel
Triggered when an user (bot included) joins a channel.
user - An IRCClient.IRCUser object specifying the user who joined the channel
channel - A string representing the joined channel
